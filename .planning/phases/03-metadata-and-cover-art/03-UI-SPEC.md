# Phase 3 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — Avalonia FluentTheme (not a React/shadcn project) |
| Preset | not applicable |
| Component library | Avalonia 11.3.12 built-ins (ItemsControl, WrapPanel, Border, ScrollViewer, Grid, StackPanel) |
| Icon library | Material.Icons.Avalonia 3.0.0 (MaterialIcon control, MaterialIconKind enum) |
| Font | Segoe UI, Inter, system-ui, -apple-system, sans-serif (declared on DesktopShell window) |

**Note — shadcn gate:** This is an Avalonia/.NET project, not React/Next.js/Vite. The shadcn initialization gate does not apply. Design tokens are maintained in AXAML resource dictionaries and inline AXAML attributes.

| Token | Value | Usage |

| sm | 8px | Compact element spacing, chip row gaps, badge padding |
| md | 16px | Default panel padding, list row horizontal padding, header bar padding |

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Caption | 12px | 400 (regular) | 1.4 | Platform badge, game count, secondary metadata, filter chip labels, search result sub-text, "No art" badge, scraping section heading |
| Body | 14px | 400 (regular) | 1.5 | List row primary text, drawer description text, metadata field values, search result count status line |

**Source:** Font sizes confirmed from DesktopShell.axaml (20/12), GameListView.axaml (20/12/16), SettingsView.axaml (28/14/13/12/11), PlatformGridView.axaml (16/12).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#1a1a2e` | Main window background, header bars, sidebar background, drawer background, content area background |
| Secondary (30%) | `#16213e` | Card/tile backgrounds, settings panel backgrounds, list row hover surfaces, drawer inner panels |
| Placeholder/Border | `#0f3460` | Cover art placeholders (until art loaded), hover state for tiles, divider borders, badge/chip backgrounds |
| Accent (10%) | `#7C6FF7` | Reserved for specific elements only — see list below |
| Muted | `#8888aa` | Secondary labels, inactive nav items, platform badge text, metadata sub-labels, filter chip X icon |
| Dim | `#555577` | Section labels (caps style), empty state body copy, hint text |
| Destructive | `#E05A5A` | Destructive actions only: remove source button, danger zone label, missing-file warning icon |

**Accent `#7C6FF7` reserved for:**
1. Active nav sidebar border (left 3px accent stripe on active nav item)
2. Primary CTA buttons — "Add ROM Source", "Scrape Metadata" (Settings panel CTA)
3. Filter chip active state fill — chips that are currently applied
4. "Enable" source action text (re-enable a disabled source)
5. Interactive search "Search globally" link/action text

Accent is NOT used on hover states, game tiles, drawer background, or read-only text.

**Platform-themed tile border (new in Phase 3):** Each game tile gains a decorative 2px border whose color is derived from the platform's theme color. Platform theme colors are defined per-platform in platforms.json (or as a fallback palette). This border replaces the uniform `#0f3460` CornerRadius border on the cover art area. The surrounding tile card background remains `#16213e`. Platform theme colors must be stored in platforms.json alongside existing alias data.

**Source:** Colors extracted from DesktopShell.axaml, GameListView.axaml, SettingsView.axaml. Accent reserved-for list from CONTEXT.md ("Filter chip visual styling inherits existing `#7C6FF7` accent, matching nav active state").

## Component Inventory (Phase 3 New Surfaces)

### 1. Game Tile — Enhanced (GameListView grid mode)

Extends existing 120×160 tile. New elements layered onto existing StackPanel+Grid:

- **Cover art Image control** (120×100): replaces `#0f3460` placeholder Border. `Stretch="UniformToFill"`, loaded async via `PathToImageConverter`. While loading: show `#0f3460` placeholder. On confirmed unmatched: show placeholder + "No art" badge.
- **Platform-themed border**: 2px solid border on the cover art area (the 120×100 region), color = platform theme color. Not on the outer tile card.
- **"No art" badge** (conditional): small pill overlay bottom-left of art area. Background `#0f3460`, text `#8888aa`, font 12px Caption, text "No art". Visible only when `IsUnmatched = true` and `CoverArtPath = null`.
- **Play button overlay** (bottom-right of art area): circular button, 44×44px, semi-transparent dark background (`rgba(0,0,0,0.6)` approximated as `#99000000`), white MaterialIcon `Play` 24×24. Visible on pointer-over and keyboard focus. Not visible at rest. Clicking dispatches launch command (emulator not wired until Phase 4 — button is present but no-ops with tooltip "Launch — emulator not configured yet").
- **Favorite heart** (top-left overlay): unchanged from Phase 2 — 28×28px button, Heart icon, `BoolToHeartColorConverter`.
- **Missing warning** (top-right overlay): unchanged from Phase 2 — 16×16 Warning icon `#E05A5A`.

### 2. Game Tile — List Mode Row (GameListView list mode)

Extends existing 56px-height row:

- **Thumbnail Image** (40×40): replaces `#0f3460` placeholder. `Stretch="UniformToFill"`, CornerRadius 4. Placeholder `#0f3460` while loading.
- Play button not shown in list mode — row click opens detail drawer.

### 3. Game Detail Drawer (new — right-side slide-in panel)

- **Trigger**: clicking any game tile (grid or list mode) slides open this drawer from the right edge of the ContentArea. Clicking the tile again or pressing Escape closes it.
- **Width**: 320px fixed. The game list behind it remains visible but is not scrollable while drawer is open (pointer-events blocked on list, not visually obscured).
- **Animation**: translate-x slide from +320px to 0 over 200ms, ease-out. No opacity fade.
- **Background**: `#1a1a2e` with left border `#0f3460` 1px.
- **Layout (top to bottom, Padding 24px all sides):**
  1. Large boxart Image: full width (272px), height 272px (1:1), `Stretch="UniformToFill"`, CornerRadius 4, `#0f3460` placeholder while loading.
  2. Title TextBlock: 20px, SemiBold, `#FFFFFF`, Margin top 16px.
  3. "Also known as:" collapsible row (Expander control or ToggleButton+StackPanel): Caption 12px `#8888aa` heading, collapsed by default, shows regional alias list when expanded.
  4. Metadata grid (2-column key/value): Genre, Year, Developer, Publisher — key at Caption 12px `#555577`, value at Body 14px `#FFFFFF`.
  5. Description TextBlock: Body 14px `#FFFFFF`, line-height 1.5, MaxLines=6, TextTrimming="CharacterEllipsis" with "Read more" toggle.
  6. Regional variants section: horizontal list of flag+ROM-name rows, Caption 12px `#8888aa`. Each row shows flag emoji + ROM filename.
  7. Region override picker: dropdown (ComboBox) for per-game preferred region. Caption label `#555577`, selected value `#FFFFFF`.
  8. Action button row (horizontal, Spacing 8):
     - **Play** (primary, accent): 44px height, Background `#7C6FF7`, CornerRadius 6, text "Play" 14px SemiBold White, Play icon 16×16 white. Full width minus Edit button.
     - **Edit** (secondary, outline): 44px height, Background Transparent, BorderBrush `#555577` 1px, CornerRadius 6, text "Edit" 14px `#8888aa`, Pencil icon 16×16 `#8888aa`. Fixed 80px width.
- **Edit mode**: triggered by Edit button. Fields become TextBox controls with `#16213e` background, `#0f3460` border. Cover art shows "Pick image..." overlay button. Save button replaces Play button (label "Save changes", accent fill). Cancel button reverts all edits.

### 4. Search Bar (new — GameListView header, second row)

Added as a second DockPanel.Dock="Top" row beneath the existing header bar.

- Background: `#1a1a2e`, Padding 8px horizontal, 8px top, 4px bottom.
- TextBox: full width minus status text area. Placeholder text "Search [Platform Name]..." at `#555577`. FontSize 14px. Background `#16213e`, BorderBrush `#0f3460`, CornerRadius 4, Height 36px. Search icon (MaterialIcon `Magnify`) inside left of TextBox at 16×16 `#8888aa`.
- Debounce: 250ms after last keystroke before filter is applied.
- Clear button: "×" appears inside right of TextBox when text is non-empty. Clicking clears and resets filter.

### 5. Search Status Bar (new — below search bar, above filter chips)

Single-line TextBlock row:

- Format: `{X} matches in {Platform Name} / {Y} total globally`
- Font: Caption 12px `#8888aa`.
- "search globally" link: inline run styled as accent `#7C6FF7`, underlined on hover. Clicking removes platform scope and shows global results.
- Padding: 4px horizontal 16px, vertical 4px.

### 6. Filter Chip Row (new — below search status bar)

Horizontal WrapPanel, Padding 8px horizontal, 4px vertical.

- **Chip anatomy**: Border, CornerRadius 16px (pill), Padding 8px horizontal 4px vertical, Height 28px.
  - Inactive chip (dropdown trigger): Background `#16213e`, BorderBrush `#0f3460`, text Caption 12px `#8888aa`, chevron-down icon 12×12 `#8888aa` right side.
  - Active chip (filter applied): Background `#7C6FF7`, BorderBrush Transparent, text Caption 12px `#FFFFFF`, "×" close icon 12×12 `#FFFFFF` right side.
- **Available filter chips**: Genre (ComboBox dropdown), Year Range (two NumericUpDown or TextBox inputs in a flyout), Developer (ComboBox dropdown).
- **"Clear filters" button**: text-only link, Caption 12px `#7C6FF7`, visible only when at least one filter is active. Positioned right-aligned in the chip row.

### 7. "Load more" Button (new — bottom of search results)

- Visible only when result count > 50 and search text is non-empty.
- Style: text-only Button, 44px height, full-width, Background Transparent, text "Load 50 more results", FontSize 14px, Foreground `#7C6FF7`.
- BorderThickness 0.

### 8. Interactive Search Dialog (new — for unmatched games)

- Triggered by "Interactive search" context menu item on an unmatched tile, or from "No art" badge click.
- Modal dialog (Avalonia Window or hosted dialog). Width 640px, Height 480px. Background `#1a1a2e`.
- Header: game filename/title, search TextBox pre-filled, Search button (accent).
- Results list: ItemsControl, each result row shows: boxart thumbnail 48×48px, title 14px `#FFFFFF`, year+platform 12px `#8888aa`, "Select" button 32px height, Background `#7C6FF7`, CornerRadius 4, text 12px White.
- On "Select": assigns scraper match to game, closes dialog, triggers background art download.
- Empty results: "No results found. Try a different search term." Caption 12px `#8888aa`.

### 9. Scraping Progress Panel (Settings — new section)

Follows existing ScanStatusBar visual language.

- Section heading: "METADATA SCRAPING" — 12px SemiBold `#555577` caps label.
- Progress row: ProgressBar (accent `#7C6FF7` fill), below it: "Scraping X / Y games" Caption 12px `#8888aa`.
- Rate-limit pause state: "Rate limit reached — resuming in Xs" Caption 12px `#8888aa` with pause icon.
- Unmatched count badge: "X games unmatched" Caption 12px `#E05A5A` — links to a filtered unmatched list.
- Re-scrape library button: "Re-scrape library" — outline style, 36px height, BorderBrush `#555577`, text `#8888aa`, icon `Sync` 16×16. Triggers global re-scrape with confirmation: "Re-scrape will overwrite auto-detected metadata. Manual overrides are preserved. Continue? [Yes, re-scrape / Cancel]"

## Interaction States

### Cover Art Loading

| State | Visual |
|-------|--------|
| Not yet loaded | `#0f3460` solid placeholder fill |
| Loading (async) | `#0f3460` placeholder, no spinner (silent background load) |
| Loaded | Image control renders art, placeholder hidden |
| Failed / unmatched | `#0f3460` placeholder + "No art" pill badge (bottom-left of art area) |
| Override (local file) | Same as loaded — uses local file path |

### Game Tile States

| State | Visual |
|-------|--------|
| Default | `#16213e` card, art or placeholder, heart (top-left), no play button |
| Pointer-over | Play button overlay appears bottom-right of art (44×44px, `#99000000` bg) |
| Keyboard focus | Same as pointer-over — play button visible |
| Favorite | Heart icon colored `#E05A5A` (existing `BoolToHeartColorConverter`) |
| Missing file | Warning icon top-right `#E05A5A` (existing `GameStatusToVisibilityConverter`) |
| Unmatched | "No art" badge visible; no play button overlay until emulator is configured |
| Selected (drawer open) | 2px accent `#7C6FF7` outer border on tile card |

### Detail Drawer States

| State | Visual |
|-------|--------|
| Closed | Not rendered / zero-width |
| Opening | Translate-x slide from +320 to 0, 200ms ease-out |
| Open — view mode | Read-only content, Play + Edit buttons |
| Open — edit mode | Editable TextBoxes, Save + Cancel buttons, Pick image overlay |
| Saving | Save button shows loading indicator (MaterialIcon `Loading` spinning, or disabled state) |

### Filter Chips

| State | Visual |
|-------|--------|
| No filter applied | Chip shows dropdown trigger, inactive style (`#16213e` bg, `#8888aa` text) |
| Filter active | Chip fills accent `#7C6FF7`, white text, "×" close icon |
| Hover on chip | Slight brightness increase (`:pointerover` style, `#1d1d40` bg for inactive, `#8a7ef9` fill for active) |

## Accessibility Constraints

- Play button overlay: minimum 44×44px touch target (Steam Deck touch — from CONTEXT.md).
- All interactive elements must have `ToolTip.Tip` set.
- Drawer must be closeable via Escape key (keyboard trap on open drawer is not acceptable).
- Filter chip "×" close icon must have accessible label via `AutomationProperties.Name="Remove [filter name] filter"`.
- Search TextBox must have `AutomationProperties.LabeledBy` pointing to a visually-hidden label or `AutomationProperties.Name="Search games"`.
