# Plan 2-02: ROM Scanner Engine

## Goal

A background scanner discovers ROMs from configured sources, detects platforms via folder name matching, cleans filenames, performs incremental add/mark-missing against the database, and computes file hashes asynchronously — all without blocking the UI.

## Context

Plan 01 delivered Game + RomFile entities, the EF Core migration, platforms.json with aliases, and System.IO.Hashing. This plan builds the core scanning engine: PlatformRegistry for folder-to-platform matching, FilenameParser for title cleaning and region extraction, M3uParser for multi-disc games, RomScannerService for incremental scan logic, FileHasher for CRC32+MD5+SHA1, RomScanHostedService as a BackgroundService wrapper, and DI registration.

The scanner uses `IRomSourceProvider.EnumerateRomsAsync()` (already implemented by `LocalRomSourceProvider` and `OsMountedRomSourceProvider`) to discover files, then processes them through the platform detection and import pipeline.

2. **Create `PlatformRegistry.cs`:**
   - Constructor takes `IEnumerable<PlatformDefinition>` (or loads from JSON path).
   - Build two dictionaries: `_byName` (lowercase Name -> PlatformDefinition) and `_byAlias` (lowercase Alias -> PlatformDefinition).
   - `Resolve(string folderName)` — case-insensitive lookup: try Name first, then Alias. Return null if no match.
   - `IsAllowedExtension(PlatformDefinition platform, string filePath)` — check if file extension is in platform's Extensions list (case-insensitive).

   - Use `JsonSerializerOptions { PropertyNameCaseInsensitive = true }` for deserialization.

3. **Create `FilenameParser.cs`:**
   - `CleanTitle(string fileNameWithoutExtension)` — use compiled Regex `\s*[\(\[][^\)\]]*[\)\]]` to strip all parenthesized/bracketed tags, then Trim.
   - `ExtractRegion(string fileNameWithoutExtension)` — use compiled Regex `[\(\[](USA|Europe|Japan|World|En|Fr|De|Es|It|Pt|Brazil|Korea|China|Asia|Australia)[,\s\)\]]` with IgnoreCase. Return first capture group or null.
   - Both methods are static.

4. **Create `M3uParser.cs`:**
   - `ParseDiscPaths(string m3uFilePath)` — read all lines, filter blank/comment lines (starting with `#`), resolve relative paths against the .m3u file's directory, return `IReadOnlyList<string>`.
   - Static method.

5. **Create `ScanUpdate.cs`:**

7. **Create `RomScannerService.cs`:**
   - Constructor injects: `YarlDbContext db`, `PlatformRegistry registry`, `IEnumerable<IRomSourceProvider> providers`.
   - `ScanAllAsync(IProgress<ScanUpdate> progress, CancellationToken ct)`:
     a. Load all enabled `RomSource` from DB.
     b. For each source, find the matching `IRomSourceProvider` via `CanHandle()`.
     c. Call `EnumerateRomsAsync()` to get all file paths.
     d. Group files by their parent folder name. For each folder, call `registry.Resolve(folderName)`.
     e. For unmatched folders, record in `unmatchedFolders` list. Skip silently.
     f. For matched folders, filter files by `IsAllowedExtension()`.
     g. **Incremental logic per source:**
        - Load existing `RomFile.FilePath` for this source into a `HashSet<string>` via `ToHashSetAsync()`.
        - For new files (not in existing set): create Game + RomFile records. Use `FilenameParser.CleanTitle()` for Title, `Path.GetFileNameWithoutExtension()` for RawTitle. Set `CreatedAt = DateTime.UtcNow`. For `.m3u` files: set `IsM3uPlaylist = true` on the RomFile, parse disc paths via `M3uParser`, create additional RomFiles for each disc with `DiscNumber` set (1-based), all linked to the same Game.
        - For removed files (in DB but not on disk): use `ExecuteUpdateAsync` to set `Status = GameStatus.Missing` on associated Games.
        - For reappeared files (on disk AND in DB with Missing status): use `ExecuteUpdateAsync` to restore `Status = GameStatus.Active`.
        - `SaveChangesAsync()` after processing each source.
     h. Report progress via `IProgress<ScanUpdate>` after each platform folder is processed.
     i. Return `ScanReport` with summary.
   - **Important:** Begin `ScanAllAsync` with `await Task.Yield()` is NOT needed here (it's in the HostedService). The scanner service itself is a straightforward async method.
   - Use `FileInfo` to get `FileSize` for each new RomFile.

8. **Create `RomScanHostedService.cs`:**
   - Inherits `BackgroundService`.
   - Constructor injects: `IServiceScopeFactory scopeFactory`, `LibraryViewModel libraryVm`.
   - `ExecuteAsync`:
     a. `await Task.Yield()` — critical to not block app startup.
     b. Create scope via `_scopeFactory.CreateScope()`.
     c. Resolve `RomScannerService` from scope (and `PlatformRegistry`).
     d. Create `Progress<ScanUpdate>` that updates `_libraryVm.ScanProgressText` and `_libraryVm.IsScanning` on UI thread (Progress<T> captures SynchronizationContext).
     e. Set `_libraryVm.IsScanning = true`.
     f. Call `scanner.ScanAllAsync(progress, stoppingToken)`.
     g. On completion, set `_libraryVm.IsScanning = false`, update status message with summary.
     h. Wrap in try/catch — log errors via Serilog, never crash the host.

1. **Create `FileHasher.cs`:**
   ```csharp
   using System.IO.Hashing;
   using System.Security.Cryptography;

   namespace YARL.Infrastructure.Scanning;

   public static class FileHasher
   {
       public static async Task<(string Crc32, string Md5, string Sha1)> ComputeHashesAsync(
           string filePath, CancellationToken ct = default)
       {
           using var crc32 = new Crc32();
           using var md5 = MD5.Create();
           using var sha1 = SHA1.Create();
           await using var stream = new FileStream(filePath, FileMode.Open, FileAccess.Read,
               FileShare.Read, bufferSize: 81920, useAsync: true);

           var buffer = new byte[81920];
           int bytesRead;
           while ((bytesRead = await stream.ReadAsync(buffer.AsMemory(0, buffer.Length), ct)) > 0)
           {
               var span = buffer.AsSpan(0, bytesRead);
               crc32.Append(span);
               md5.TransformBlock(buffer, 0, bytesRead, null, 0);
               sha1.TransformBlock(buffer, 0, bytesRead, null, 0);
           }
           md5.TransformFinalBlock([], 0, 0);
           sha1.TransformFinalBlock([], 0, 0);

           return (
               crc32.GetCurrentHashAsUInt32().ToString("X8"),
               Convert.ToHexString(md5.Hash!),
               Convert.ToHexString(sha1.Hash!)
           );
       }
   }
   ```
   Key: Use `GetCurrentHashAsUInt32().ToString("X8")` for CRC32, NOT `Convert.ToHexString(GetCurrentHash())` — the latter produces wrong endianness (RESEARCH.md Pitfall 6).

2. **Modify `RomScannerService.cs`:** After `SaveChangesAsync()` for new records, add a background hashing pass:
   - Query all RomFiles for this source where `CRC32Hash == null` and `IsM3uPlaylist == false`.
   - Use `Parallel.ForEachAsync` with `MaxDegreeOfParallelism = 2` (Claude's discretion — avoids saturating disk I/O).
   - For each file, call `FileHasher.ComputeHashesAsync()`, then update the RomFile record.
   - Use a separate DbContext scope for hash updates (create a new scope inside the parallel loop to avoid thread-safety issues with EF Core).
   - Alternatively, collect results and batch-update after parallel completion to keep it simple with a single DbContext.
   - Wrap individual file hashing in try/catch — skip files that are locked or inaccessible, log warning.
