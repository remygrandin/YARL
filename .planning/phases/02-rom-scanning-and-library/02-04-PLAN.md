# Plan 2-04: Library UI Views (Avalonia AXAML)

## Goal

The desktop shell displays a fully functional library UI: left sidebar navigation, platform grid with tiles, game list within a platform (grid + list modes), recently played carousel, favorites carousel, scan progress status bar, empty state with CTA, and an Add ROM Source dialog — all bound to LibraryViewModel via compiled bindings and styled per the UI-SPEC dark theme.

## Context

Plan 03 delivered a fully wired LibraryViewModel with DynamicData SourceCache, PlatformViewModel, GameViewModel, derived collections (Platforms, AllGames, Favorites, RecentlyPlayed), scan state, and commands. This plan creates the AXAML views that bind to those ViewModel properties.

All views must use compiled Avalonia bindings (`x:DataType` declarations) as `AvaloniaUseCompiledBindingsByDefault=true` is set in the csproj. The `Material.Icons.Avalonia` NuGet package must be added for icons. Dark theme colors from UI-SPEC: `#1a1a2e` (background), `#16213e` (card surfaces), `#0f3460` (hover/active), `#7C6FF7` (accent).

### Task 2-04-02: Visual Inspection Checkpoint

