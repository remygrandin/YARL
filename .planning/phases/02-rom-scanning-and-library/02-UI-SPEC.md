# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: ROM Scanning and Library.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Avalonia FluentTheme — not a web project) |
| Preset | not applicable |
| Component library | Avalonia FluentTheme (dark variant) — App.axaml confirmed |
| Icon library | Material.Icons.Avalonia (NuGet: `Material.Icons.Avalonia`) — user decision |
| Font | Segoe UI on Windows / system-ui (Inter fallback) on Linux — Avalonia system font stack |

**Note:** shadcn gate does not apply. This is an Avalonia/.NET desktop application, not a React/Next.js/Vite project. The design system is Avalonia FluentTheme with a dark color override layer.

---

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, badge inner padding, disc number inline gap |
| sm | 8px | Carousel item spacing, game tile inner padding, status bar padding, icon button gaps |
| md | 16px | Card inner padding, sidebar item padding, dialog field spacing, section content padding |
| lg | 24px | Section header margin-bottom, carousel section vertical padding, dialog section spacing |
| xl | 32px | Top-level content area padding (left/right), between carousel sections |
| 2xl | 48px | Major layout region separation (sidebar + content gap not applicable — sidebar is fixed) |
| 3xl | 64px | Empty state vertical centering offset |

Exceptions:
- Sidebar width: 120px fixed (not a spacing token — structural layout value, user decision)
- Controller focus ring offset: 2px border + 4px outline-offset (= 6px total visual gap from element edge)
- Touch/controller minimum target: 44px height for all interactive elements in fullscreen shell
- Carousel tile size: 120px wide × 160px tall (game tiles) — from RESEARCH.md Pattern 8
- Platform grid tile size: 180px wide × 200px tall — user decision
- Status bar height: 32px fixed

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 24px | 700 (Bold) | 1.2 | Shell header "YARL", empty state heading, dialog title |
| Heading | 20px | 600 (SemiBold) | 1.2 | Section labels ("Continue Playing", "Favorites", platform name in grid tile), sidebar section headers |
| Body | 16px | 400 (Regular) | 1.5 | Game titles in list view, status bar text, dialog body copy, scan progress text |
| Caption | 12px | 400 (Regular) | 1.4 | ROM count badge on platform tile, platform badge on game tile, region tag, file size, carousel item subtitle |

Font weight scale is exactly two weights: 400 (Regular) and the Bold/SemiBold pair treated as one display weight group (600 for section heads, 700 for shell-level display only). Executor must not introduce intermediate weights (500, 300).

Font family declaration for AXAML:
```
FontFamily="Segoe UI, Inter, system-ui, -apple-system, sans-serif"
```
Apply at the root Window element of both DesktopShell and FullscreenShell so all child elements inherit.

## Color

| Role | Hex | Usage |
|------|-----|-------|
| Dominant (60%) | `#1a1a2e` | Window background, sidebar background, dialog overlay backdrop base, fullscreen background — confirmed from FullscreenShell stub |
| Secondary (30%) | `#16213e` | Card surfaces (platform tiles, game tiles, carousel items, dialog body, status bar background) |
| Tertiary surface | `#0f3460` | Hover state on tiles and sidebar nav items, selected/active sidebar item background |
| Accent (10%) | `#7C6FF7` | Focus rings, primary CTA buttons, active sidebar nav indicator strip, favorite heart icon (filled), scan spinner, dialog confirm button — user decision |
| Text primary | `#FFFFFF` | Headings, game titles, platform names, sidebar labels (active) |
| Text secondary | `#ccccdd` | Body text, status bar text, dialog body — from FullscreenShell stub |
| Text muted | `#8888aa` | Captions, ROM count, platform badges, sidebar labels (inactive) — from FullscreenShell stub |
| Destructive | `#E05A5A` | Destructive actions only: "Remove ROM Source" confirm button, no other use |
| Missing overlay | `#1a1a2e` at 50% opacity | Greyed-out overlay on game tiles with `Status = Missing` |

Accent `#7C6FF7` is reserved for:
1. Keyboard/controller focus ring (2px border + 4px glow)
2. Primary CTA buttons (Add ROM Source, Confirm in dialogs)
3. Active sidebar nav indicator (3px left border strip)
4. Favorite heart icon when IsFavorite = true
5. Scan progress spinner
6. Selected item highlight border on platform/game tiles

Accent must NOT be used for: general hover states, text color, decorative borders, section dividers.

## Surface Inventory

Eight UI surfaces for this phase. Each surface section specifies exact dimensions, layout, states, and AXAML control choices.

### Surface 1: Platform Grid View

**Purpose:** Main library view — shows all platforms with ≥1 scanned ROM as portrait card tiles.

**Layout:**
- Control: `ItemsRepeater` with `UniformGridLayout` (not WrapPanel — enables virtualization for large collections)
- Tile size: 180px wide × 200px tall — user decision
- Tile gap: 16px (md) horizontal and vertical
- Content area padding: 32px (xl) left/right, 24px (lg) top

**Platform Tile anatomy (180×200):**
```
┌──────────────────────┐  ← Border, radius 8px, background #16213e
│                      │
│   [Platform Logo]    │  ← Image, 140×100, centered, top padding 20px
│   or [Color+Name     │    Fallback: Rectangle filled with platform color,
│        text block]   │    platform name centered in white 14px Bold
│                      │
│  Platform Name       │  ← TextBlock, 16px SemiBold, #FFFFFF, center-aligned
│                      │    Margin: 8px top from logo
│  142 games           │  ← TextBlock, 12px Regular, #8888aa, center-aligned
│                      │    Margin: 4px top from name
└──────────────────────┘
```

**States:**
- Default: background `#16213e`, no border
- Hover (mouse): background `#0f3460`, cursor pointer
- Controller focus: 2px solid `#7C6FF7` border, box-shadow equivalent (Avalonia: `BoxShadow="0 0 0 4 #7C6FF780"`)
- Pressed: background `#0f3460` at 80% opacity, scale 0.97 (AvaloniaUI ScaleTransform)

**Interaction:** Single click/A-button navigates to Game List for that platform.

**Filtering rule:** Platforms with 0 games are excluded from the Items source in the ViewModel (not hidden via Visibility — they must not be in the collection at all).

### Surface 2: Game List (Within a Platform)

**Purpose:** Shows all games for a selected platform. Switchable between Grid mode and List mode.

**Header bar (above game list):**
- Platform name: 20px SemiBold, `#FFFFFF`
- ROM count: 12px Regular, `#8888aa`, inline after platform name
- View toggle: two icon buttons (GridView icon / ListView icon from Material.Icons.Avalonia), right-aligned in header bar
- Favorites filter toggle: heart icon button, right of view toggle, toggles between "all games" and "favorites only"
- Back button: chevron-left icon, 16px, left of platform name, returns to Platform Grid

**Grid mode:**
- Control: `ItemsRepeater` with `UniformGridLayout`
- Tile size: 120px wide × 160px tall (same as carousel tiles for visual consistency)
- Gap: 8px (sm)
- Tile anatomy: identical to carousel tile (see Surface 4)

**List mode:**
- Control: `ItemsRepeater` with `StackLayout` (vertical)
- Row height: 56px fixed
- Row layout: [Platform logo thumbnail 40×40, radius 4px] [Game title 16px Regular, #FFFFFF, leading] [Platform badge 12px, #8888aa, trailing] [Favorite heart icon 20px, trailing]
- Row padding: 16px left/right, 8px top/bottom
- Row hover: background `#0f3460`
- Divider: 1px `#0f3460` between rows (not needed on hover row)

**Missing game state (both modes):**
- Tile/row receives 50% opacity overlay (Avalonia: `Opacity="0.5"` on the root Border)
- Warning triangle icon (Material.Icons.Avalonia: `Warning`) overlaid top-right of tile, 16px, `#E05A5A`
- Tooltip on hover/focus: "File not found — reconnect your ROM source to restore"

**Interaction (Grid):** Single click/A-button opens game detail (Phase 4 — stub in Phase 2).
**Interaction (List):** Single click/A-button same. Favorite toggle: click heart or Y-button.

### Surface 3: Flat List View

**Purpose:** Alternative to Platform Grid — all games in one scrollable list, filterable by platform.

**Access:** Sidebar nav item "Library" with sub-toggle, or dedicated sidebar "All Games" entry (see Surface layout).

**Platform filter bar:**
- Horizontal ScrollViewer + StackPanel(Horizontal) above the game list
- Filter chips: each platform that has ≥1 game renders a chip
- Chip anatomy: platform name text 12px + game count badge 10px, height 28px, horizontal padding 12px, radius 14px (pill shape)
- Chip default: background `#16213e`, text `#ccccdd`
- Chip selected: background `#7C6FF7`, text `#FFFFFF`
- "All" chip always first
- Chips scroll horizontally; no wrapping

**Game list:**
- Same list-mode row anatomy as Surface 2 List mode
- Platform badge is always visible (shows which platform each game belongs to)
- Sorted alphabetically by title by default

**Empty filter state:** If active filter chip returns 0 games: centered text "No games for this platform yet." 16px Regular `#ccccdd`.

### Surface 4: Recently Played Carousel

**Purpose:** Horizontal row at top of home/library view showing last 10–15 played games.

**Section header:** "Continue Playing" — 20px SemiBold, `#FFFFFF`, margin-bottom 8px (sm).
Show section only when `RecentlyPlayed.Count > 0`. When empty (no games launched yet), section is hidden entirely — do not show an empty row.

**Implementation:** `ScrollViewer` with `HorizontalScrollBarVisibility="Auto"` + `VerticalScrollBarVisibility="Disabled"` wrapping `ItemsControl` with `StackPanel(Horizontal, Spacing="8")`.

**Game tile anatomy (120×160):**
```
┌────────────────┐  ← Border, radius 6px, background #16213e
│                │
│  [Art/Logo]    │  ← Image or platform-color Rectangle, fills 120×120
│                │
│  Game Title    │  ← TextBlock, 12px Regular, #FFFFFF, MaxLines=2, TextTrimming=CharacterEllipsis
│  [PlatBadge]   │  ← TextBlock, 10px Regular, #8888aa
└────────────────┘
```

**States:**
- Default: background `#16213e`
- Hover: background `#0f3460`
- Controller focus: 2px `#7C6FF7` border + glow

**Scrollbar:** Visible on hover (desktop). On fullscreen/controller: scrollbar hidden; left/right D-pad navigates between tiles.

### Surface 5: Favorites Carousel

**Purpose:** Horizontal row below Recently Played, showing all favorited games — user decision: position is below "Continue Playing".

**Section header:** "Favorites" — 20px SemiBold, `#FFFFFF`, margin-bottom 8px (sm). Heart icon (Material.Icons.Avalonia: `Heart`, 20px, `#7C6FF7`) inline left of label text.

Show section only when `Favorites.Count > 0`. When no games are favorited, section is hidden — do not show an empty row or "no favorites" message here (empty state is handled by the library's empty state surface for the no-source case only; zero favorites is normal and silent).

**Implementation and tile anatomy:** Identical to Surface 4 (Recently Played Carousel). Same control structure, same tile dimensions, same states.

**Vertical spacing between carousel sections:** 32px (xl) between the bottom of Recently Played tiles and the "Favorites" section header.

### Surface 6: Scan Progress Status Bar

**Purpose:** Persistent feedback strip at the bottom of the window during active scanning. Disappears when scan is complete (after 3-second hold on summary message).

**Dimensions:** Full window width × 32px fixed height. Docked to `DockPanel.Dock="Bottom"`.

**Background:** `#16213e` (secondary surface).

**Layout (left to right inside the 32px bar):**
- 8px left padding
- Spinner icon (Material.Icons.Avalonia: `Sync`, 16px, `#7C6FF7`, rotation animation 360°/1s linear infinite) — visible only when `IsScanning = true`
- 8px gap
- Status text (16px Regular, `#ccccdd`): "Scanning SNES... 142 games found" — bound to `LibraryViewModel.ScanProgressText`
- Flexible spacer
- Cancel button (text "Cancel", 12px Regular, `#8888aa`, no border, hover underline) — visible only when `IsScanning = true`. Triggers `CancelScanCommand`.
- 8px right padding

**States:**
- Scanning: spinner visible, Cancel visible, text = current platform progress
- Complete: spinner hidden, Cancel hidden, text = "Scan complete — 1,847 games" for 3 seconds, then bar collapses (Height = 0 with 200ms height animation)
- Idle (no scan ever run): bar not rendered (collapsed, Height = 0)

**Transition:** Height animates from 32px to 0 using Avalonia animation (not instant collapse). Duration 200ms, easing Linear.

### Surface 7: Add ROM Source Dialog

**Purpose:** User enters a folder path and selects the source type (Local or OS-Mounted). Triggered by "Add ROM Source" CTA.

**Container:** Avalonia `Window` (modal dialog, not an overlay panel) — `Width="480"`, `Height="auto"` (min 280px), `SizeToContent="Height"`, `WindowStartupLocation="CenterOwner"`.

**Background:** `#1a1a2e`. Dialog chrome border: 1px `#0f3460`.

**Layout (vertical stack, padding 24px all sides):**

1. **Title row:** "Add ROM Source" — 24px Bold, `#FFFFFF`. Close button (Material.Icons.Avalonia: `Close`, 20px, `#8888aa`) right-aligned in same row.

2. **Divider:** 1px `#0f3460`, margin 16px top/bottom.

3. **Path field:**
   - Label: "Folder Path" — 12px Regular, `#8888aa`, margin-bottom 4px
   - TextBox: full width, height 40px, background `#16213e`, border 1px `#0f3460`, border-radius 6px, text 16px Regular `#FFFFFF`, padding 8px left/right
   - Focus border: `#7C6FF7` 1px, no glow on TextBox (glow is for tile focus rings only)
   - Browse button: "Browse..." text button, right-aligned below the TextBox, 12px Regular `#8888aa`, margin-top 4px. Opens Avalonia `OpenFolderDialog`.

4. **Source type field:** (margin-top 16px)
   - Label: "Source Type" — 12px Regular, `#8888aa`, margin-bottom 4px
   - Two radio buttons stacked vertically, gap 8px:
     - "Local Folder" (default selected)
     - "OS-Mounted (network share)"
   - Radio button label: 16px Regular `#FFFFFF`

5. **Validation message area:** height 20px, reserved below source type. Text 12px Regular `#E05A5A`. Shown when path field is empty on submit attempt: "Please enter a folder path."

6. **Divider:** 1px `#0f3460`, margin 16px top/bottom.

7. **Action row:** right-aligned, gap 8px.
   - Cancel button: height 36px, min-width 80px, background `#16213e`, border 1px `#0f3460`, radius 6px, label "Cancel" 14px Regular `#ccccdd`.
   - Add Source button: height 36px, min-width 120px, background `#7C6FF7`, no border, radius 6px, label "Add Source" 14px SemiBold `#FFFFFF`. Disabled state: background `#7C6FF780` (50% opacity), cursor default.

**Dismiss behavior:** Cancel button, Close button, and Escape key all close without saving. Clicking outside the dialog does NOT dismiss (modal).

**Submit behavior:** "Add Source" button validates path is non-empty, then calls `AddRomSourceCommand`. On success dialog closes. On error (path not found): validation message shows "Folder not found. Check the path and try again."

**Purpose:** Shown in the main content area when no ROM sources have been configured yet.

**Layout:** Vertically and horizontally centered in the content area (right of sidebar in desktop shell; full window in fullscreen shell).

```

        browsing your collection.

```

**Vertical spacing:** Icon → heading: 24px (lg). Heading → body text: 8px (sm). Body text → CTA button: 32px (xl).

**CTA button "Add ROM Source":**
- Height: 44px, min-width: 200px, radius: 8px
- Background: `#7C6FF7`, label: "Add ROM Source" 16px SemiBold `#FFFFFF`
- Leading icon: Material.Icons.Avalonia `FolderPlus`, 20px, `#FFFFFF`, gap 8px to label
- Hover: background `#6A5FE0` (10% darker)
- Focus ring: 2px `#7C6FF7` + glow (same as tile focus ring)

- This state is visible from app launch until the first scan completes and at least one platform tile is available to render.

## Controller Focus Rules

Applies to FullscreenShell and any screen reachable from it. Desktop shell mouse interaction takes precedence; controller navigation is additive.

- D-pad Left/Right: navigate between tiles within a carousel row or grid row
- D-pad Up/Down: move between carousel sections / grid rows / sidebar items
- A: confirm / select / open
- B: back / cancel
- Y: toggle favorite on focused game tile
- X: toggle grid/list view mode (within Game List screen only)
- Focus wrap: carousel rows wrap at ends (left from first tile jumps to last; right from last wraps to first)
- Focus trap: Add ROM Source Dialog traps focus — Tab cycles through fields, Escape dismisses

Focus ring style (all focusable elements in fullscreen shell):
```xml
FocusAdorner: Border with BorderBrush="#7C6FF7" BorderThickness="2"
              BoxShadow="0 0 0 4 #7C6FF780"
```
