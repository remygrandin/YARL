# Plan 2-03: Library ViewModel + DynamicData Live Collections

## Goal

LibraryViewModel is the live data hub: a DynamicData `SourceCache<GameViewModel, int>` receives scanner results, derived collections for platforms/favorites/recently-played update reactively, and scan state (progress, running, rescan command) is fully wired — enabling the UI layer (Plan 04) to bind without any data logic.

## Context

Plan 02 delivered the scanner engine (RomScannerService, RomScanHostedService, PlatformRegistry, FileHasher). The scanner currently updates `LibraryViewModel.ScanProgressText` and `IsScanning` via IProgress. This plan expands LibraryViewModel with DynamicData SourceCache, creates PlatformViewModel and GameViewModel wrapper classes, wires the scanner's database results into the SourceCache, and exposes derived collections (platforms with game counts, favorites, recently played).

DynamicData is already a transitive dependency via ReactiveUI.Avalonia — no new package needed.

**Implementation:**

1. **Create `GameViewModel.cs`:**
   ```csharp
   using ReactiveUI;
   using ReactiveUI.SourceGenerators;
   using YARL.Domain.Enums;
   using YARL.Domain.Models;

   namespace YARL.UI.ViewModels;

   public partial class GameViewModel : ReactiveObject
   {
       public int Id { get; }
       public string Title { get; }
       public string RawTitle { get; }
       public string PlatformId { get; }
       public string? Region { get; }
       public GameStatus Status { get; }
       public DateTime CreatedAt { get; }

       [Reactive] private bool _isFavorite;
       [Reactive] private DateTime? _lastPlayedAt;

       public GameViewModel(Game game)
       {
           Id = game.Id;
           Title = game.Title;
           RawTitle = game.RawTitle;
           PlatformId = game.PlatformId;
           Region = game.Region;
           Status = game.Status;
           CreatedAt = game.CreatedAt;
           _isFavorite = game.IsFavorite;
           _lastPlayedAt = game.LastPlayedAt;
       }
   }
   ```
   - Class must be `partial` for `[Reactive]` source generator.
   - `IsFavorite` and `LastPlayedAt` are mutable (togglable by user).
   - All other properties are read-only (set from Game entity).

2. **Create `PlatformViewModel.cs`:**

3. **Rewrite `LibraryViewModel.cs`:**
   The existing stub has only `StatusMessage`. Replace entirely with a full implementation:
   ```csharp
   using System.Collections.ObjectModel;
   using System.Reactive.Disposables;
   using System.Reactive.Linq;
   using DynamicData;
   using ReactiveUI;
   using ReactiveUI.SourceGenerators;
   using YARL.Domain.Enums;
   using YARL.Domain.Models;
   using YARL.Infrastructure.Scanning;

   namespace YARL.UI.ViewModels;

   public partial class LibraryViewModel : ReactiveObject, IDisposable
   {
       private readonly CompositeDisposable _disposables = new();
       private readonly SourceCache<GameViewModel, int> _gamesSource = new(g => g.Id);

       // Bound to UI
       private ReadOnlyObservableCollection<GameViewModel> _allGames = new([]);
       public ReadOnlyObservableCollection<GameViewModel> AllGames => _allGames;

       private ReadOnlyObservableCollection<PlatformViewModel> _platforms = new([]);
       public ReadOnlyObservableCollection<PlatformViewModel> Platforms => _platforms;

       private ReadOnlyObservableCollection<GameViewModel> _recentlyPlayed = new([]);
       public ReadOnlyObservableCollection<GameViewModel> RecentlyPlayed => _recentlyPlayed;

       private ReadOnlyObservableCollection<GameViewModel> _favorites = new([]);
       public ReadOnlyObservableCollection<GameViewModel> Favorites => _favorites;

       // Scan state
       [Reactive] private string _statusMessage = "YARL is ready. Add ROM folders to get started.";
       [Reactive] private bool _isScanning;
       [Reactive] private string _scanProgressText = "";

       // Selected state
       [Reactive] private PlatformViewModel? _selectedPlatform;

       // Commands
       public ReactiveCommand<Unit, Unit> RescanCommand { get; }
       public ReactiveCommand<GameViewModel, Unit> ToggleFavoriteCommand { get; }

       // Scanner reference for rescan triggering
       private readonly IServiceScopeFactory? _scopeFactory;
       private readonly PlatformRegistry? _platformRegistry;
       private CancellationTokenSource? _scanCts;
       public ReactiveCommand<Unit, Unit> CancelScanCommand { get; }
   ```

   Constructor wiring:
   - Accept `PlatformRegistry` and `IServiceScopeFactory` via DI (nullable for unit tests).
   - **AllGames pipeline:**
     ```csharp
     _gamesSource.Connect()
         .ObserveOn(RxApp.MainThreadScheduler)
         .Bind(out _allGames)
         .Subscribe()
         .DisposeWith(_disposables);
     ```
   - **Platforms pipeline:** Use `_gamesSource.Connect()` -> `GroupOn(g => g.PlatformId)` -> transform each group into a `PlatformViewModel` with the group count. Use `AutoRefreshOnObservable` to keep counts updated. Platforms with 0 games are excluded (filter Count > 0). Bind to `_platforms`.
   - **RecentlyPlayed pipeline:** `_gamesSource.Connect()` -> `AutoRefresh(g => g.LastPlayedAt)` -> `Filter(g => g.LastPlayedAt != null)` -> `SortBy(g => g.LastPlayedAt, SortDirection.Descending)` -> `Top(15)` -> `ObserveOn(RxApp.MainThreadScheduler)` -> `Bind(out _recentlyPlayed)`.
   - **Favorites pipeline:** `_gamesSource.Connect()` -> `AutoRefresh(g => g.IsFavorite)` -> `Filter(g => g.IsFavorite)` -> `ObserveOn(RxApp.MainThreadScheduler)` -> `Bind(out _favorites)`.
   - **ToggleFavoriteCommand:** `ReactiveCommand.Create<GameViewModel>(async gvm => { gvm.IsFavorite = !gvm.IsFavorite; /* persist to DB via scope */ })`.
   - **RescanCommand:** `ReactiveCommand.CreateFromTask(async () => { /* create scope, resolve scanner, run ScanAllAsync, reload SourceCache */ })`, canExecute: `this.WhenAnyValue(x => x.IsScanning).Select(s => !s)`.
   - **CancelScanCommand:** `ReactiveCommand.Create(() => _scanCts?.Cancel())`, canExecute: `this.WhenAnyValue(x => x.IsScanning)`.

   Public methods:
   - `LoadGamesFromDb(YarlDbContext db)` — load all Active games from DB, convert to GameViewModel, call `_gamesSource.AddOrUpdate()` for each. Called by RomScanHostedService after scan.
   - `AddOrUpdateGame(GameViewModel gvm)` — public accessor for SourceCache AddOrUpdate (used by scanner to push live updates during scan).
   - `Dispose()` — dispose `_disposables` and `_scanCts`.

4. **Modify DI registration in `Program.cs`:** Update LibraryViewModel registration to inject dependencies:

5. **Modify `RomScanHostedService.cs`:** After scan completes, call `_libraryVm.LoadGamesFromDb(db)` to populate the SourceCache from database. During scan, push live updates via `_libraryVm.AddOrUpdateGame()` for each newly created game (convert Game entity to GameViewModel).
